## What is monkmade-tools?

monkmade-tools is a collection of small, useful software tools published under the Monkmade name.

It is not currently treated as a classic SaaS product.

## Why does this project exist?

The main purpose is distribution.

monkmade-tools exists to help build audience, learn how to publish, test what people actually use, and create public proof that useful tools are being shipped.

## Current goal

Reach the first 100 real users.

Success is measured by real usage, feedback, shares, comments, downloads, and repeat interest — not by code volume or feature count.

## How to work on this project

Prefer small, shippable improvements.

Prioritize:

- Make the tool usable.
- Make it easy to understand.
- Make it easy to share.
- Prefer publishing over polishing.
- Learn from real user signals.

Avoid overengineering.

Do not turn this into a large SaaS platform unless there is clear external demand.

## Tool selection

New tools should be small, useful, easy to explain, and easy to publish.

Prefer tools that:

- solve a clear, everyday problem
- can be understood within seconds
- can be used without an account
- can be shared easily
- provide visible value quickly
- are simple enough to build and maintain solo

Avoid tools that:

- require a platform, dashboard, account system, or complex backend
- need heavy ongoing support
- are mainly interesting because of technical complexity
- do not help reach real users
- turn monkmade-tools into a SaaS product

Before building a new tool, ask:

Can this tool realistically help attract or serve the next real user?

If not, do not build it.

## Voice-/Transkript-Schutz

Michel arbeitet teilweise mit VoiceRecord und fügt längere transkribierte Texte ins Terminal ein.

Transkripte können falsch erkannte Wörter, abgeschnittene Sätze, doppelte Satzteile oder zufällige Fremdphrasen enthalten.

Wenn eine Anweisung widersprüchlich wirkt, fachlich nicht zu monkmade-tools passt, wie ein Transkriptfehler klingt oder riskante Dateiänderungen auslösen würde, nicht raten und nicht blind handeln.

Stattdessen kurz nachfragen und die wahrscheinlich gemeinte Bedeutung nennen.

Harmlos erkennbare Tipp- oder Transkriptfehler dürfen still sinnvoll interpretiert werden.

## Decision rule

Before adding complexity, ask:

Would this help get the next real user?

If not, keep it simple.
