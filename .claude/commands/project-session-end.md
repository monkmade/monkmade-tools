# /project-session-end

Ziel: Eine Projekt-Session sauber abschließen.

## Ablauf

1. Fasse kurz zusammen:
   - Was wurde geändert?
   - Was wurde entschieden?
   - Was funktioniert?
   - Was ist offen?
   - Was ist der nächste sichere Schritt?
2. Prüfe, ob Build/Test sinnvoll oder nötig ist.
3. Prüfe, ob Git-Status genannt werden sollte.
4. Aktualisiere `docs/PROJECT_STATUS.md`.
5. Sage Michel, ob noch etwas vor dem Schließen offen ist.

## No-Gos

- Nicht automatisch committen.
- Nicht neue Aufgaben anfangen.
- Nicht große Refactors starten.
